🚁 琛光E1 Mini 无人机开发与编程课程

从焊接到飞控源码 · 从零打造你的ESP32无人机

ESP32 开源无人机 | Arduino + Python 双语言教学
40总课时
1.5h单节时长
6教学模块
60h总学时

📋 课程概览

本课程以 琛光E1 Mini ESP32开源无人机 为教学平台，专为初一学生设计。 每名学生将亲手从零打造一台属于自己的无人机——从PCB焊接、电机安装，到用Arduino修改飞控源码（CF-Drone），再到用Python通过MAVLink协议实现外部编程控制。 课程贯穿「造出来→飞起来→改代码→写程序」完整链路。

⏱ 每节课（90分钟）标准时间分配：
① 课前回顾与导入（10分钟）→ ② 核心知识讲解（15-25分钟）→ ③ 动手实践/编程（40-50分钟）→ ④ 讨论/调试/展示（12分钟）→ ⑤ 总结与课后任务（8分钟）

🔧 平台详解

📋 硬件规格

主控芯片ESP32 DevKit V1（30Pin，双核240MHz，内置Wi-Fi+蓝牙）
姿态传感器MPU-6500 六轴陀螺仪+加速度计（I2C接口）
电机8520空心杯电机 × 4（3-3.7V，约4-5万转/分）
螺旋桨65mm/66mm/75mm（对角同向，正反桨配对）
电池1S航模锂聚合物电池（3.7-4.2V，高放电倍率）
供电管理ME6118A33PG LDO稳压 + 470μF滤波电容
PCB嘉立创打样，1.6mm厚度，拼板设计（主控板+电机座+支架）

💻 软件与固件

飞控固件CF-Drone（Arduino开发，开源 GitHub: songge8/CF-Drone）
固件源码CF-Drone.ino(主程序) / imu.ino(传感器) / control.ino(飞行控制) / pid.h(PID算法) / motors.ino(电机) / web_rc.ino(网页遥控) / cli.ino(调试命令) / mavlink.ino(MAVLink) / wifi.ino / battery.ino / safety.ino 等
控制方式①网页遥控器（手机连Wi-Fi「Drone_WIFI」→ 浏览器 http://192.168.4.1:8080）
控制方式②QGC App（MAVLink协议，Ground Control Station）
控制方式③SBUS实体遥控器（主板预留3P接口，GPIO4）
控制方式④Python外部编程（pymavlink库，通过Wi-Fi发送MAVLink指令）
CLI调试命令ca(校准陀螺仪) / ps(查看状态) / mfl-mfr-mrl-mrr(电机测试) / cr(遥控校准)
开发工具Arduino IDE（固件开发）+ Python 3.x（外部控制）

💡 教学建议与注意事项

⚠️ 1. 安全三原则（不可妥协）
①焊接时佩戴护目镜，烙铁放回支架确认稳固
②电池充电必须有人在场，鼓包/发热立即隔离停用
③调试阶段必须摘掉所有桨叶；飞行时安装桨叶保护罩；飞行高度不超过50cm；教师随时准备切断电源
2. 焊接教学的关键
第7课（焊接练习）是最关键的入门课。建议准备充足的废板/洞洞板供练习，不要直接在E1 Mini PCB上焊第一焊。先练手法，再焊真机。每位学生焊完后必须用万用表逐点检测，通过后才可通电。
3. 备件准备
电机和桨叶是易损件，建议多备50%的8520电机和桨叶。ESP32开发板建议多备2-3块。建议准备1-2台教师预装好的E1 Mini作为「标准参考机」，用于对比调试。
4. Arduino C语言教学策略
初一学生学C语言有一定挑战。教学策略：①每个概念都配无人机相关案例；②每节课留出充足上机编程时间；③善用串口监视器作为即时反馈工具；④对进度慢的学生提供课后练习和一对一辅导。
5. 源码阅读策略
CF-Drone固件对学生来说是第一个大型真实项目源码。教学策略：①先看文件名猜功能→再看代码确认；②不要逐行讲解，抓住关键函数和数据流；③用「找宝藏」游戏化方式；④修改从小处着手，建立信心后再深入。
6. 课程弹性调整建议
如果学生编程基础较好，可压缩模块四至6节课（合并23-24、25-26），将多出的2节课分配给模块五。如果焊接进度慢，可适当延长模块二至12课。模块五（固件开发）是核心特色，不建议压缩。
